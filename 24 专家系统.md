专家系统 = 推理引擎 + 知识
类似于，程序=数据结构 + 算法

知识是专家系统的核心，首先需要领域内的专家，构造知识库
这些专家拥有知识和经验，他们输出自己的经验，
知识工程师把专家的经验转换成计算机系统能够识别的知识的格式
转换后，所有的知识存储到了知识库中
系统工程师或算法工程师，让专家系统拥有一定的AI能力，可以做一定的决策
他们会负责推理引擎的部分

知识图谱工程师的作用是把专家的知识经验以更好的方式组织在知识库（图数据库）

NLP工程师的作用是，在知识图谱数据库的基础上，构建推理层，包括很多算法

然后再在API层，提供业务层可以调用的接口

最后在业务层调用AI系统的接口，完成相关任务

专家 => 知识工程师 => NLP工程师 => API => 业务场景

## 特点
1. 可以处理不确定性
2. 知识的表示，图数据库，知识图谱
3. 可解释性，AI系统给出一个结果的同时需要给出这样判断的原因
4. 可以做知识推理

深度学习的最大缺点是可解释性非常差


## 知识推理
作用是根据已有知识，发现新的知识


## 缺点
1. 设计大量的规则
2. 需要领域专家来主导
3. 可移植性差
4. 学习能力差
5. 人能考虑的范围是有限的
